BigQA — Retrieval-Augmented Generation

A full RAG architecture for querying large volumes of text. Combines semantic document retrieval via HuggingFace embeddings (all-MiniLM-L6-v2), generative LLM access through OpenRouter (Qwen, Gemini, GPT), and an intuitive Streamlit interface. Automatically indexes the Ono-Enzo/Dataset_test dataset. Deployed live on HuggingFace Spaces.

Python LangChain HuggingFace OpenRouter Streamlit RAG Vector Search
GitHub Live Demo

How It Works

1
Ingest — Loads dataset from HuggingFace, parses and chunks documents automatically.
2
Embed — Generates semantic vectors using all-MiniLM-L6-v2 for similarity search.
3
Retrieve — On user query, finds the most relevant document chunks via vector search.
4
Generate — Sends retrieved context + question to LLM via OpenRouter for precise answer generation.
5
Display — Streamlit UI shows answers with source references for full transparency.

Weather ETL — Automated Data Pipeline

Complete ETL pipeline for real-time weather data from São Paulo. Collects data from the OpenWeatherMap API every hour, transforms into structured format via pandas, and stores in MySQL. Fully containerized with Docker and orchestrated by Apache Airflow. Includes extraction validation, timestamp normalization, and data quality checks.

Python Apache Airflow Docker MySQL pandas Redis OpenWeatherMap
GitHub →

Pipeline Stages

E
Extract — HTTP GET to OpenWeatherMap API, validates status, saves raw JSON.
T
Transform — Normalizes nested JSON, drops unnecessary columns, converts Unix timestamps to timezone-aware datetime (America/São_Paulo), saves as Parquet.
L
Load — Connects to MySQL via SQLAlchemy, appends to sp_weather table, validates row count.
⚙
Orchestrate — Airflow DAG runs hourly (0 */1 * * *), with Redis as message broker, scheduler, worker, and triggerer containers.
